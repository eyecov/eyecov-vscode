# Covflux VS Code Extension

Display code coverage from Covflux SQLite database or **coverage-json** folder directly in your VS Code editor.

## Features

- **Automatic Coverage Display**: Shows coverage for files as you open them
- **Gutter Indicators**: Visual indicators in the editor gutter showing which lines are covered
- **Real-time Updates**: Automatically refreshes when the coverage database or coverage-json files are updated
- **SQLite Integration**: Reads directly from the Covflux SQLite database
- **Coverage-JSON**: Optional support for the coverage-json folder format (mirrors codebase with `_coverage.json` and `files/*.json`)

## Requirements

- VS Code 1.74.0 or higher
- A Covflux SQLite database file (generated by `covflux-core`), **or** a coverage-json folder

## Extension Settings

- `covflux.databasePath`: Path to the Covflux SQLite database file (default: `${workspaceFolder}/coverage.sqlite`)
- `covflux.coverageJsonPath`: Path to the coverage-json folder (default: `${workspaceFolder}/coverage-json`)
- `covflux.source`: Which source to use: `sqlite`, `coverage-json`, or `auto` (try SQLite first, then coverage-json)
- `covflux.coverageJsonStripPathPrefix`: Strip this prefix from paths when resolving coverage-json (e.g. `app` for Laravel so `app/Http/...` → `Http/...`). Default: `app`. Set to empty to disable.
- `covflux.debug`: When `true`, write debug logs to View → Output → Covflux (default: `false`).
- `covflux.showCoverageOnOpen`: Automatically show coverage when files are opened (default: `true`)
- `covflux.showUncovered`: Highlight uncovered lines (default: `true`)
- `covflux.showCovered`: Highlight covered lines (default: `true`)
- `covflux.showLineCoverage`: Show coverage by changing background color of lines (default: `true`)
- `covflux.showGutterCoverage`: Show coverage with gutter icons in addition to background colors (default: `false`)

## Commands

- `Covflux: Show Coverage` - Manually enable coverage display
- `Covflux: Hide Coverage` - Manually disable coverage display
- `Covflux: Toggle Coverage` - Toggle coverage display on/off
- `Covflux: Show Coverage Info` - Show detailed coverage information for the current file

## Status Bar

The extension adds two status bar indicators:

1. **Coverage Status** (right side) - Shows coverage percentage for the current file (e.g., "85.3% (213/250)"). Color-coded:
   - Green: ≥80% coverage
   - Yellow: 50-79% coverage
   - Red: <50% coverage
   - Click to toggle coverage display

2. **Database Status** - Shows database connection status:
   - ✓ "Connected" - Database is connected and working
   - ✗ "No DB" - Database file not found
   - ✗ "DB Error" - Error connecting to database

## Debugging

To verify the extension is working:

1. Check the status bar for database connection status (or "Coverage-JSON" when using coverage-json).
2. Open a file with coverage data — you should see coverage percentage in the status bar.
3. **View → Output → select "Covflux"** to see which source was used, how many uncovered lines were loaded, and how many decoration ranges were applied. If `uncoveredSet` is 0 or `uncovered` ranges are 0, check path resolution and that the coverage-json folder layout matches your source tree.
4. Optionally check the Developer Console for logs starting with `[Covflux]` that show:
   - Database connection status
   - Coverage data loading for files
   - Errors if any occur

To view the Developer Console:
- Press `Cmd+Shift+P` (Mac) or `Ctrl+Shift+P` (Windows/Linux)
- Type "Toggle Developer Tools"
- Check the Console tab for `[Covflux]` logs

## Usage

1. Ensure you have a Covflux SQLite database file generated by `covflux-core`
2. Open your workspace in VS Code
3. The extension will automatically detect and display coverage for files with coverage data
4. Coverage indicators will appear in the gutter next to covered/uncovered lines

## Installation

### Option 1: Install from VSIX Package (Recommended)

1. **Build the extension:**
   ```bash
   npm install
   npm run compile
   ```

2. **Package the extension:**
   ```bash
   npm install -g vsce  # If you don't have vsce installed
   npm run package
   ```
   This creates a `.vsix` file (e.g., `covflux-vscode-0.1.0.vsix`)

3. **Install the VSIX:**

   **Using Command Line:**
   ```bash
   code --install-extension covflux-vscode-0.1.0.vsix
   ```

   **Using VS Code GUI:**
   - Open VS Code
   - Go to Extensions view (`Ctrl+Shift+X` / `Cmd+Shift+X`)
   - Click the `...` menu (top right)
   - Select "Install from VSIX..."
   - Choose the `.vsix` file

4. **Reload VS Code** to activate the extension

### Option 2: Development Mode (For Testing)

1. **Open the extension folder in VS Code:**
   ```bash
   code /path/to/covflux-vscode
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Compile:**
   ```bash
   npm run compile
   ```

4. **Press `F5`** to launch a new Extension Development Host window

   The extension will run in the Development Host window. You can set breakpoints, check console logs, and test features.

### Option 3: Link Locally (Advanced)

1. Build and compile as above
2. Use symlinks to the extension folder (not recommended for production)
